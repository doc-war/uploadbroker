Upload Broker

轻量级资源代理网关 —— 专为 AI 推理平台的临时文件上传、读取与生命周期管理设计。不重新发明存储，而是做现有存储（Local / S3 / R2 / OSS）的统一安全外壳。

它就是临时存储界的 Redis。

Why Upload Broker

三大核心开源价值

不是又一个对象存储，而是 AI 推理时代的资源代理基础设施。

⚡

AI 推理时代的刚需基础设施

首个为AI中间件服务的文件Broker。推理场景中，临时图片、PDF、语音、视频文件高速流转。Upload Broker 填补了"AI 临时文件生命周期管理"的空白 —— 就像存储界的 Redis，专注缓存、临时、高性能，而非底层持久化。

🧱

极致的防腐层

业务从本地存储切到阿里云 OSS 再到 Cloudflare R2，上层 Platform 一行代码都不用改。Platform 只需要无脑调用 POST /upload，拿回标准 URL。底层怎么切、怎么换驱动，业务层零感知。

🔐

渐进式安全架构

可选的 hmac_secret 让项目能屈能伸：

个人开发者：不加 Secret，直接当个人图床或测试工具，零配置启动。
企业用户：开启 HMAC + 多 Salt 轮换，直接接入微服务矩阵。

同一份代码，在"吸引初学者"和"满足企业落地"之间取得完美平衡。

Quick Start

30 秒启动

单二进制，零外部依赖。

1

下载二进制（或 go install）

2

编写配置 uploadBroker.yaml

3

启动 Broker

4

上传文件

5

读取文件

# uploadBroker.yaml
listen: 127.0.0.1:9001
base_url: https://upload.example.com
url_blake2b_salts:
  - current-salt
url_prefix: tmp

storage:
  upload_driver: local
  drivers:
    local:
      root: ./objects

# optional hmac_secret for production
# hmac_secret: "platform_broker_shared_token"
$ ./uploadBroker
$ curl -F "file=@photo.png" "http://127.0.0.1:9001/v1/upload"
{"code":0,"data":{"url":"https://...","mimeType":"image/png"}}
$ curl "https://upload.example.com/tmp/..." # 读取文件
