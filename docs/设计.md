# Upload Broker API 设计规范

## 1. 概述

Upload Broker 是一个轻量级资源代理（Resource Broker），负责临时资源的上传、读取、生命周期管理以及存储抽象，为 AI 推理平台提供统一的资源访问能力。

Upload Broker 不面向终端用户，仅作为 Platform 的本机服务运行，由 Platform 通过 localhost 调用。

职责边界：

> **Platform 决定谁可以访问（Who）；Broker 决定资源是否合法以及如何获取（What & How）。**

Broker 不负责：

- 用户认证
- API Key 管理
- Project 管理
- 配额、限流、计费
- CDN、业务路由

以上均属于 Platform。

------

## 2. 设计原则

- 单二进制部署（Go）
- Broker 与 Platform 解耦
- HTTP API 统一读写接口
- 内容寻址（Content Addressable）
- 临时资源（TTL）
- 存储驱动可插拔（Local / R2 / S3 / OSS）
- URL 协议由 Broker 独占实现
- Platform 不解析 URL 协议

------

```text

    │

```

## 4. Object Model

每个资源对应一个 Object。

| 字段        | 说明                 |
| ----------- | -------------------- |
| key         | BLAKE2b-256 内容哈希 |
| mime_type   | MIME 类型            |
| size        | 文件大小             |
| backend     | 存储驱动             |
| storage_key | Driver 对象定位标识  |
| created_at  | 创建时间             |
| expire_at   | 过期时间             |

原则：

- key 永久不变
- URL 可以变化
- storage_key 仅 Driver 可见

------

## 5. Upload API

```http
POST /v1/upload
```

Content-Type：

```text
multipart/form-data
```

参数：

| 参数    | 必填 | 说明              |
| ------- | ---- | ----------------- |
| file    | 是   | 上传文件          |
| expires | 否   | 1~24 小时，默认24 |

支持资源类型：

| 类型         | MIME                                                   | 默认最大体积 |
| ------------ | ------------------------------------------------------ | ------------ |
| 图片         | image/png、image/jpeg、image/webp                      | 2 MB         |
| 音频         | audio/mpeg、audio/mp3、audio/wav、audio/mp4、audio/aac | 3 MB         |
| 视频（预留） | video/mp4、video/webm                                  | 10 MB        |

Broker 完成：

- MIME 检测
- Hash 计算
- TTL 计算
- Storage 写入
- SQLite 写入
- URL 签发

成功返回：

```json
{
  "code": 0,
  "data": {
    "url": "https://upload.xxx.com/tmp/...",
    "mimeType": "image/png",
    "size": 12345,
    "expireAt": "2026-06-28T00:00:00Z"
  }
}
```

BaseURL 为 Broker 启动配置项。

------

## 6. Read API

```http
GET /tmp/{expire}/{shard}/{key}/{token}.{ext}
```

Broker 收到请求后执行：

1. URL 格式校验
2. expire 校验
3. token 校验
4. Object 查询
5. Driver Get()
6. 返回文件流

Platform 不参与 URL 解析，仅负责认证、日志、统计等外围能力。

------

## 7. URL 协议

统一格式：

```text
/tmp/{expire}/{shard}/{key}/{token}.{ext}
```

说明：

| 字段   | 含义            |
| ------ | --------------- |
| expire | Unix 秒级时间   |
| shard  | key 前2位       |
| key    | 内容哈希        |
| token  | URL 签名        |
| ext    | MIME 推导扩展名 |

URL 由 Broker 生成。

URL 仅由 Broker 验证。

Platform 不解析任何字段。

URL 签名算法固定为 **BLAKE2b-256**，仅允许配置 Salt，不允许修改算法。

生成 URL 时使用第一个 Salt；校验 URL 时按顺序尝试所有 Salt（最多两个），用于平滑轮换。

------

## 8. Storage Driver

统一接口：

```go
type Storage interface {
    Put(ctx,obj)
    Get(ctx,storageKey)
    Delete(ctx,storageKey)
    Health(ctx)
}
```

`storage_key` 为 Driver 私有对象定位标识，由 Driver 生成并写入 SQLite；`key` 为内容标识，二者职责不同。

职责：

- 保存对象
- 获取对象
- 删除对象
- 健康检查

Driver 不负责：

- URL
- TTL
- Token
- SQLite
- HTTP

------

## 9. 元数据存储

使用 SQLite。

仅保存元数据。

开启：

- WAL
- busy_timeout
- 单写多读

Broker 所有状态均来自 SQLite。

其中：

```text
storage_key

↓

20260627/4d/db913be8....png
```

用于 Driver 定位实际对象。

------

## 10. Local Driver

本地 Driver 使用日期目录组织文件。

目录结构：

```text
{root}/{yyyyMMdd}/{shard}/{key}.{ext}
```

例如：

```text
./objects/
└──20260627/
    └──4d/
        db913be8....png
```

其中：

- yyyyMMdd：创建日期
- shard：key 前两位
- key：内容哈希

SQLite 保存对应的 `storage_key`，读取时由 Driver 根据 `storage_key` 定位文件。

------

## 11. 生命周期

```text
Upload
    │
Available
    │
TTL
    │
Expired
    │
Cleanup
    │
Delete
```

默认 TTL：

24 小时。

后台定时删除：

- Storage Object
- SQLite Record

## 12. 配置

启动时可以通过`uploadBroker --config=/path/to/config.yaml`指定配置文件路径加载配置。

```yaml
listen: 127.0.0.1:9001

# 可选。若不为空，则 upload 接口强制校验请求参数sign和timestamp
hmac_secret: "platform_broker_shared_secure_token_abc123"

# 必须配置
base_url: https://upload.example.com
url_blake2b_salts:
  - current_salt
  - previous_salt
url_prefix: tmp   # 可选，默认tmp

# 可选，以下都是默认值
metadata_db: ./data/broker.db
cleanup_interval: 10m
default_ttl: 24h
limits:
  image: 2MB
  audio: 3MB
  video: 10MB

# 存储驱动
storage:
  upload_driver: local
  drivers:
    local:
      root: ./objects
```
说明
| 配置项                | 说明                                                       |
| --------------------- | ---------------------------------------------------------- |
| listen                | Broker 监听地址                                            |
| hmac_secret           | 用于计算HMAC-SHA256(hmac_secret, timestampt)，进行上传校验 |
| base_url              | 对外资源 URL 前缀                                          |
| metadata_db           | SQLite 数据库                                              |
| url_prefix            | URL 路径前缀                                               |
| url_blake2b_salts     | URL 签名 Salt，最多两个，用于平滑轮换                      |
| default_ttl           | 默认资源有效期                                             |
| cleanup_interval      | 后台清理周期                                               |
| storage.upload_driver | 上传使用的存储驱动                                         |
| storage.drivers       | Storage Driver 配置                                        |
| limits                | 各资源类型大小限制                                         |

## 13. Health API

```http
GET /v1/health
```

返回 Storage Driver `Health()` 检测结果与 SQLite 连接状态，供运维及负载均衡探活。

示例：

```json
{
  "version": "1.0.0",
  "status": "ok",
  "storage": "ok",
  "sqlite": "ok",
  "time": "2026-06-27T12:00:00Z"
}
```

------

## 14. 安全要求

- Broker 仅部署于内网
- MIME 实际检测
- 内容寻址命名
- URL 签名校验
- 路径穿越保护
- 文件大小限制
- TTL 自动失效
- URL 只读

## 16. 非目标

Upload Broker 不提供：

- 用户系统
- 权限管理
- Project 管理
- API Key
- 配额管理
- 图片处理
- 文件浏览
- 长期对象存储
- CDN 管理
- 多租户能力

其唯一职责是：

> 提供统一、安全、可过期的资源上传与读取能力，并屏蔽底层存储实现，为上层 Platform 提供稳定一致的 Resource API。
